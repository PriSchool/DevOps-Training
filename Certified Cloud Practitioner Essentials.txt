---------------------
Basic Infrastructure
---------------------

Amazon EC2
שירות המיחשוב הבסיסי המוצע, שרץ על שרתים פיזיים בחוות שרתים של אמאזון, שמאפשרים גישה ושליטה על מערכות הפעלה מלאות (ווינדוס או לינוקס).

Amazon Elastic Load Balancer - ELB
מערכת שיוצרת סדר וחלוקה נכונה של אינפוט בקשות באופן אוטומטי; יוצרת איזון בין כל סך הבקשות הנכנסות למערכת.

Amazon EC2 Auto Scaling
מערכת ששולטת באופן אוטומטי על השימוש בשרתים שאמאזון מציעה, בצורה כזו שלא מצריכה מהמשתמש להחזיק בשרתים שלא נמצאים בשימוש פעיל, או להחזיק בשרתים בכמות שלא מספקת לו לרגע הנוכחי.
המערכת משתמשת בכמות המדוייקת של כוח מחשוב פיזי הנצרך בכל רגע נתון על ידי המשתמש.

Amazon Simple Notification Service - SNS
מערכת שמאפשרת שליחה של מידע לקבוצה גדולה של נמענים, על פי שיטת פאב/סאב.
pub/sub, publish/subscribe

Amazon Simple Queue Service - SQS
מערכת שמאפשרת הפרדה בין שולח בקשה לבין הנמען על ידי יצירת תור, או באפר ביניהם.
promotes de-coupling, which improves instances of one failure causing the entire system to collapse.

AWS Lambda 
שירות שפועל על שרתים, אך לא מצריך שליטה, גישה או טיפול פעיל על ידי המשתמש באופן קבוע - נקרא serverless.
מתאים למערכות שצריכות לבצע חישובים בזמן שלא עולה על רבע שעה.

Amazon Elastic Container Service - ECS
מערכת לטיפול ושימוש בקונטיינרים, ספציפית בדוקרים.
מאפשרת לשלוט בקלאסטרים של דוקרים שונים, שצריכים לפעול יחד בהרמוניה כשמעוניינים לשלוט על כמה קונטיינרים בו בזמן.

Amazon Elastic Kubernetes Service - EKS
עובד בצורה דומה ל-ECS, אבל עם מערכת Kubernetes.

AWS Fargate
אם מעוניינים להריץ ECS או EKS על שרתים אך אין צורך בשליטה או גישה למערכת ההפעלה עצמה בזמן הפיתוח, אין צורך להשתמש ב-EC2 הקלאסי, ולכן אפשר להשתמש בשירות serverless שנקרא Fargate.

----------------------
Global Infrastructure
----------------------

AWS CloudFront 
כלי המשמש כמעין שירות cache המאפשר להעביר מידע למרחקים גדולים על פני כדור הארץ בצורה יעילה יותר, על ידי שימוש ב-edge locations, במקום להמשיך לשלוח מידע ממרחקים גדולים בצורה קבועה.

AWS Outposts
האפשרות לקשר בין on-premises לבין שירותי AWS העולמיים; מעין קישור היברידי שמביא שרתים פיזיים on-premises כדי לשמור על latency נמוך.

-----------------
AWS Connectivity
-----------------

AWS Managment Console / AWS CLI / AWS SDKs
דרכים שונות להשתמש בפועל בשירותי AWS, על ידי גישה דרך דפדפן, דרך קוד low level בשימוש command line, או בקוד high level דרך שפות תכנות שונות בשימוש ב-SDK המאפשרים זאת.

AWS Elastic Beanstalk
שירות שמאפשר לקחת קוד קיים, וממנו ליצור באופן אוטומטי את כל תהליכי ושירותי EC2 הרצויים, כולל הקונפיגורציות השונות ושמירתן לשימוש חוזר עתידי - מנהל עבורנו את כל ענייני load balancing, capacity adjusting, automatic scaling, applicaion health monitoring. מאפשר להתמקד בצד העסקי מאשר בצד הניהולי.

AWS CloudFormation
שירות שמאפשר ליצור טמפלייט מסויים של הגדרת עבודה עם resources שונים של AWS בלי להגדיר באופן מדוייק במה אתה בוחר להשתמש, ומשם המערכת נבנית באופן אוטומטי על ידי CloudFormation - בין היתר עבור שירותי EC2, machine learning, analytics, storage. מאפשר להתייחס לעבודה מול שירותי AWS והשרתים עצמם בקוד במקום להשתמש באחת משיטות הגישה הרגילות כמו managment console.

-----------
NETWORKING
-----------

AWS VPC - Virtual Private Cloud
היכולת ליצור אזור תחום, פרטי, שיכול להיות מקושר לשאר הרשת או לא, ויוצר הפרדה מסויימת כמו סאבנטים.

AWS Direct Connect
מאפשר ליצור קישור ישיר, עם latency נמוך ואבטחה ברמה גבוהה ע"י חיבור fiber. כך ניתן להתחבר ל-VPC בצורה מרוחקת, פרטית, תוך כדי שמירה על חיבור מהיר ומאובטח. 

--------
STORAGE
--------

Amazon Elastic Block Store - EBS 
זיכרון קשיח שמאפשר לשמור מידע בין הפעלות EC2 שונות, שכן כל EC2 הוא מכונה וירטואלית וכיבוי של אחד והפעלה של אחר כוללת איבוד של מידע "זמני" שמאוחסן על גבי זיכרון volatile הנקרא instance store.
EBS גורם לכך שמידע יישמר בין הפעלות שונות, אך בנוסף כדאי להפעיל snapshots כל כמה זמן, כדי ליצור מעין שמירות גיבוי נוכחיות של המידע הקיים, בכדי למזער נזקים בעת איבוד מידע.
לא ניתן להגדלה באופן דינאמי, וחייב להיות באותו availability-zone של ה-EC2 שאליו הוא מקושר. בייסיקלי כונן קשיח.

Amazon Simple Storage Service - S3
שירות אחסון מידע, שומר קבצים של עד 5 טרה במקומות אחסון שנקראים buckets. מגיע במספר שירותים בדרגות שונות כגון:
S3 Standard; שומר מספר עותקים של אותו מדיע במספר מקומות, בכדי ליצור גיבוי. בעל עמידות גבוהה ביותר לאחסון המידע ושמירתו לטווח ארוך.
S3 Infrequent Access, S3-IA; משומש במקרים בהם המידע השמור לא צריך להיפתח לעיתים קרובות, אבל ברגע שכן הואא נדרש במהירות גבוהה.
S3 Glacier; משומש במקרים בהם המידע השמור לא צריך להיפתח במהירות. אפשר בניהול השוטף להכניס קבצים לתוך vaults ולתת להן הגבלות וחוקים לגבי המידע השמור בהן (הרשאות, זמני פתיחה..).
בכל המקרים, ניתן להעביר בין הדרגות השונות באופן אוטומטי (30 ימים פה, ואז 60 ימים פה..).
S3 מתאים למקרים בהם מתעסקים עם אובייקטים שלמים, או כשלא מבצעים שינויים בקבצים לעיתים קרובות.

Amazon Elastic File System - EFS
managed file system, כזו שמאפשרת לגשת ולשלוט במידע מכל EC2 בתוך אותו region (מספר availability-zones), בצורה סקאלאבילית ביחס לגודל הזיכרון השמור.
דומה ל-EBS אבל לא מוגבל בגודל המידע השמור בו, ולא משוייך ל-EC2 מסויים בזמן העבודה מולו.

---------
DATABASES
---------

AWS Aurora
מערכת לניהול בסיסי נתונים, שנשמרת ומגובה במספר עותקים על גבי S3; מסוגל לעבוד עם MySQL או עם PostgreSQL.

Amazon Relational Database Service - RDS 
מערכת לשמירה וניהול בסיסי נתונים בענן, בצורה כזו שמנהלת באופן אוטומטי את "מאחורי הקלעים" של ניהול בסיסי נתונים - תומכת ברוב המערכות לניהול בסיסי נתונים.

Amazon DynamoDB
non-relational, NoSQL database, כזה שמאפשר עבודה בעזרת queries פשוטים שלא משתמשים ב-SQL ומאפשרים שליטה ברמת טבלה בודדת שמסוגלת להחזיק אובייקטים שונים זה מזה.
highly-scalable, מהיר מאוד לשימוש.

Amazon DynamoDB Accelerator - DAX
שכבת caching עבור בסיסי נתונים NoSQL המייעלת מהירויות גישה לנתונים.

Amazon Redshift
מערכת לאחסון ושליטה בנתונים "בהסתכלות לאחור", מה שנקרא data warehouse; דומה לבסיסי נתונים רגילים אבל מתעסק בנתונים שכבר "חקוקים בסלע", כאלה של ישתנו יותר (אנליטיקה של big data..)

AWS Database Migration Service - DMS 
שירות שמאפשר העברת אחסון מידע לתוך שירותי AWS מתוך בסיסי נתונים קיימים ששמורים כרגע on-premises או בענן.
עונה על downtime מזערי, שמאפשר להמשיך לעבוד עם תוכנות שמסתמכות על בסיסי הנתונים הנוכחיים.
בהנחה ובסיס הנתונים המקורי והחדש הם מאותו טייפ, אין בעיה בהעברה. במידה והם שונים, יש צורך להעביר את המידע דרך AWS Schema Convertion Tool שמאפשר לגשר בין השניים.
משמש גם ל-database consolidation, העיקרון של שילוב מספר בסיסי נתונים לבסיס נתונים יחיד.

Amazon DocumentDB
אם יש צורך בשמירת נתונים בשיטת NoSQL, אך מעוניינים לשמור יותר מאשר מספר attributes קטנים לכל key. 
בשירות זה, מתממשק עם MongoDB, ומשמש לשמירת נתונים כגון קונטנט נרחב, פרופילים, קטלוגים שלמים.

Amazon Neptune
בסיס נתונים גרפי, העונה על הקושי הטמון בבסיסי נתונים מסורתיים, שקשה לעקוב ולנהל בהם אחר רשתות שלמות של קישורים בין ישויות.

Amazon Quantum Ledger Databse - QLDB
בסיס נתונים שמידע בו נשאר immutable - לעולם לא יכול להשתנות או להימחק מהרשומות.

Amazon ElastiCache
מוסיף שכבות caching על פני בסיסי הנתונים, בכדי לייעל את מהירויות הגישה למידע השמור.

--------
SECURITY
--------

Shared Responsibility Model
הרעיון שעל פיו ישנה חלוקה ברמות האבטחה הקשורות בחומרה ובתוכנה המשתתפות בכל תהליך של AWS.
לדוגמה, במערכת EC2 יחידה, ישנה הפרדה בין האבטחה ש-AWS עצמה אחראית עלייה (השרתים הפיזיים, הבניין בו השרתים שמורים ומוגנים, הרשת שמחוברת ל-instance של אותו EC2 וכו'), לעומת האבטחה שהלקוח \ המשתמש של אותה יחידת EC2 אחראי עלייה (מערכת ההפעלה הספציפית, התוכנות המתוקנות, הדאטא השמור עצמו).

AWS Identity and Access Management - IAM
מערכת האבטחה הבסיסית שיוצרת משתמשים חדשים ומתאחלת אותם כך שאין בידם הרשאה אפילו לרכיב אחד במערכת, כולל התחברות ראשונית.
בתור המשתמש הראשי במערכת, אתה נקרא root account user, ובידך כל ההרשאות לכלל חלקי המערכת.
אפשר ליצור IAM groups ובכך ליצור קבוצות בעלות הרשאות שונות, ולתוכן לחלק את משתמשי המערכת.
ניתן גם ליצור roles, "תפקידים" מסויימים שמאפשרים גישה או מונעים גישה למשתמש בעל תפקיד מסויים בהיררכיה, ומטבעו role הוא זמני ומשתנה. בכך ניתן לשלוט על הרשאות שונות באאופן דינאמי.

AWS Organizations
מערכת שמאפשרת לשלוט במשתמשי AWS שונים - רמת אבסטרקציה נוספת על פני שימוש במשתמש יחיד שאחראי על הכול.

AWS Shield
מגיע בגרסת Standard לכל משתמשי AWS - מגן מפני סוגי מתקפות DDoS נפוצות.
ישנה גם גרסת Advanced, המעניקה אפשרויות נוספות ואינטגרציה עם שאר מערכות AWS.

Amazon Inspector
שירות המריץ בדיקה על החלקים השונים המרכיבים את מערכת AWS בידי המשתמש, ומציגה חולשות אופציונליות, פגיעויות ודרכי התמודדות איתן.

----------------------
MONITORING & ANALYTICS
----------------------

Amazon CloudWatch
שירות ניטור מידע real-time לגבי האפליקציות והחומרה שקשורים בשירותי AWS שאתה משתמש בהם.
ניתן גם לשלוח הודעות על ידי שירות SNS בהתאם ל-thresholds שנקבעו מראש.

AWS CloudTrail
API auditing tool, משמש למעקב אחר כל פעולה שמגיעה במגע עם AWS, כולל מידע על מי שלח את הבקשה, איך הוא ביצע אותה, מתי, הרשאות, פעולות ספציפיות..

AWS Trusted Advisor
שירות אוטומטי של AWS שבודק מספר תחומים הקשורים בשימוש השוטף של המשתמש בשירותי AWS. התחומים כוללים:
cost optimization, performance, security, fault tolerance, service limits.
השירות נותן הצעות לשיפור במספר התחומים המצויינים, על פי תוכניות העבודה השונות מול AWS.

EXTRAS
------

AWS Snow Family
היכולת לרתום משאבי AWS לטובת migration של בסיסי נתונים שלמים on-premises אל תוך שירותי AWS.
כוללת 3 אופציות עיקריות-
AWS Snowcone, המסוגל להחזיק עד 8 טרהבייט. פיזית נשלח הלוך וחזור מ-AWS על מנת להתחבר ל-EC2 של המשתמש.
AWS Snowball, מסוגל להחזיק עד 80 טרהבייט בגרסת storage optimized, או עד 42 טרהבייט בגרסת compute optimized; משמש לטובת חיבור למערכת קיימת כך שניתן להריץ כבר ממנו Lambda לדוגמה.
AWS Snowmobile, משאית עצומה שמסוגלת להכיל על 100 פטהבייט; מתחברת on-premises ואז מעבירה את כל המידע הנדרש.

Amazon SageMaker
כלי לניהול ופיתוח כלים הקשורים בלמידת מכונה.
